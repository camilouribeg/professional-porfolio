Extensibility

One framework, any industry

The plugin architecture makes this platform sector-agnostic. Extending to a new industry — mining, manufacturing, agriculture, transport — follows the same five steps every time. The core engine never changes.

1

Regulatory Mapping

Identify applicable standards, IPCC source categories, and available emission factors. Gap analysis before writing a line of code.

2

Data Ingestion

Map sector-specific data sources — mine management systems, blast records, SCADA — and build ingestion connectors.

3

EF Registry Extension

Add sector emission factors with version tags. Annual regulatory updates flow through without touching calculation code.

4

Calculation Module

Write one new plugin module with the same interface as all existing modules. Sector-specific logic stays isolated.

5

Validation & Output

Sector-specific data quality rules. Every output traceable to input version, EF version, and calculation method used.
